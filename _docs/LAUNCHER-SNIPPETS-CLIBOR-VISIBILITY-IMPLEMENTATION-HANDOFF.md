## 検証（機械的な完了判定）

1. 定型文タブ・履歴タブの両方に行区切りの罫線が表示される（目視）
2. 数字キー1〜0・Shift+1〜0での貼り付けが全行で従来どおり動く（回帰確認）
3. 検索Editに文字を打って絞り込み、絞り込み後の表示行に対する数字キー選択が正しく動く（`ResolveSnipRow`経由の回帰確認）
4. クリック選択・マウスホバーのハイライト・右クリックメニューが無変化であること
5. 白化バグが再発しないこと（ランチャーを複数回開閉して確認）

reality-checkerへの委任を推奨（コードを書いた本人が自己採点しない、というこのプロジェクトの一貫方針）。

## 地雷（設計書G節から転記、および実装で判明した追加の地雷）

- 白化バグの再燃領域（`WS_EX_COMPOSITED`）には一切触れないこと。
- **訂正**: 当初「今回の変更は生成時オプション文字列のみでその領域と無関係」と想定していたが、実際は`+Grid`が`FillLauncherSnippetsLV()`の`-Redraw`/`+Redraw`トグルと干渉し、「項目は存在するが文字が見えない」不具合を引き起こすことが実機検証で判明した。`+Grid`を含むListViewの動的再構築(`Delete`+`Add`)には、`+Redraw`だけでなく明示的な`InvalidateRect`が必要。同様のパターンを他のListViewに適用する際はこの地雷を踏まえること。
- `+Grid`はセル間の罫線のみで行の背景色は変わらない（ゼブラ配色ではない）。それは今回のスコープ外（設計書F節で不採用と判断済み）。

## 対象外（今回やらないこと・理由は設計書F節）

- 番号キーのアルファベット拡張（a〜z）: 検索Editが既定フォーカスのため不採用
- 行の背景色によるゼブラ配色（NM_CUSTOMDRAW / Class_LV_Colors.ahk）: 白化バグ再燃リスクとコストに見合わないため不採用
- サブタブ/グループ別ビュー: グループプレフィックス機能を廃止した過去の判断と矛盾するため不採用

## ビルド・反映の注意（前回セッションでの実際のつまずき）

**`dist/soushin-suggest.ahk`を編集しただけではアプリの動作は変わらない。** `dist/soushin-suggest.exe`はコンパイル済みバイナリで、ソースを直しても自動的には再ビルドされない。反映するには:

1. 稼働中の`soushin-suggest.exe`を終了する（`tasklist`で確認、`taskkill`または通常のトレイ右クリック→終了）
2. `powershell -NoProfile -ExecutionPolicy Bypass -File scripts/build.ps1 -Version <バージョン>`でビルド
3. ビルドが失敗する場合、大抵は「稼働中のexeがファイルをロックしている」ため。手順1に戻る
4. `dist/soushin-suggest.exe`を起動し直して動作確認

## 検証（設計書G節末尾の検証シナリオを実機で確認すること）

1. 複数行テキストを履歴から右クリック→整形→「改行を除去して1行に」→ 1行になって貼り付けられる。履歴の原文は無変更・変換結果は履歴に**現れない**。
2. 同じ項目で変換→「ひらがな→カタカナ」をShift押しながら選択 → 貼り付けは起きず、変換結果が履歴の**先頭に現れる**。
3. 「全角→半角」で`ＡＢＣ１２３ガガ`→`ABC123ｶﾞｶﾞ`（文字数増を確認＝2段呼び出しが正しい証拠）。
4. **整形サブメニューの1番目と6番目が異なる結果を出すことを必ず確認**（G-1のクロージャ束縛バグが実装ミスで再発していないかの検証。AutoHotkey v2のforループ変数はクロージャに参照捕捉されるため、`MakeTransformHandler`ファクトリを経由しないと全項目が同じ関数になる）。
5. 画像履歴項目の右クリック → 整形/変換メニューが**出ない**。
6. 管理画面で本文Editに入力中に^+F → 検索Editへフォーカスが移る。数字入力は奪われない。
7. （第2歩）複数行テキストを「改行ごとに履歴に展開」→ 各行が個別の履歴項目として先頭付近に現れる。101行以上のテキストでは中止メッセージが出る。
8. （第3歩）複数行テキストで「改行ごとにFIFO貼り付け」開始→対象アプリでCtrl+Vを繰り返す→1行目から順に貼られる。外部で別テキストをコピーするとモードが自動終了する。

## 地雷（設計書G節から転記、特に重要なもの）

- **G-1（最重要・新規）**: AutoHotkey v2の`for`ループ変数はクロージャに参照捕捉される。`MakeTransformHandler`ファクトリ経由で束縛を固定しないと、メニュー12項目全てが最後の変換関数になる。実装後は必ず検証シナリオ4を確認すること。
- **G-2**: 既存のメニュー表示中タイマー制御（`CheckLauncherFocus`の停止/再開）の枠組みに触らない。サブメニューは`Menu()`を追加するだけで済む。
- **G-3**: GUIコントロールの追加・スタイル変更は一切しない（今回の3ステップはメニュー・ホットキー・グローバル変数のみで完結する設計）。`WS_EX_COMPOSITED`の再提案は絶対禁止。
- **G-5**: `LCMapStringW`はLCID `0x0411`明示・2段呼び出し必須（1段目で必要バッファ長を取得）。
- **G-7**: 履歴展開は「先に`CloseLauncher()`→逆順Push→上限100行」の3点セットを外さない。
- **G-8**: FIFO/LIFOの装填遅延（`~^v`後200ms）は実機で貼り付けの遅いアプリ（リモートデスクトップ等）でも競合しないか確認し、必要なら300-500msに調整すること。

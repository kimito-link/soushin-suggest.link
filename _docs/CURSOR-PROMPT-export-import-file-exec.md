## 【最重要】スコープの境界線（絶対に超えないこと）

この実装には明確な境界線があります。設計の結論として、以下は**明確に却下**されています。

- **クリップボード履歴のエクスポート・インポートは実装しません。** 履歴は今後もディスクに一切書き込まれません。「履歴も保存できたら便利では」という発想が浮かんでも、絶対に実装しないでください。これは製品に残る最後の安全設計です
- **PowerToys Runのようなファイル検索機能は実装しません。** 「ファイル名の一部を入力して検索する」ような機能を作らないでください
- 実装するのは、**(1) 定型文（snippets.ini）のインポート・エクスポート導線**と、**(2) クリップボード履歴の中身が実在するローカルファイル/フォルダのパスだった場合に、それを開く機能**の2つだけです

## 実装1: 定型文のエクスポート・インポート

エクスポートは新機能を作りません。既存の`snippets.ini`は既にプレーンテキストファイルなので、それ自体がエクスポート形式です。トレイメニューに以下の3項目を追加してください（設計書E-1のコードをそのまま使用）。

1. 「定型文ファイルを編集 (snippets.ini)」— メモ帳で開く
2. 「定型文をインポート（別ファイルから追記）」— ファイル選択ダイアログで別のiniファイルを選び、既存の`snippets.ini`に追記する
3. 「設定フォルダを開く」— エクスプローラーで設定フォルダを開く

インポート機能（設計書E-2）は、既存の`LoadSnippets()`関数にオプションのパス引数を追加して、別ファイルの読み込みにも使えるようにしてから、`ImportSnippets()`という新しい関数を実装してください。**重要**: インポートは「既存にないラベルだけを追記する」形にしてください。既存の定型文を上書きしてはいけません。

## 実装2: クリップボード履歴からのファイル・フォルダを開く機能

これは「検索」ではありません。**「クリップボード履歴に入っているテキストが、たまたま実在するファイルやフォルダの絶対パスだったら、それを開けるようにする」**という、非常に限定的な機能です。

設計書E-3のコード（`RunnablePathFrom`関数と`OpenHistoryPath`関数）をそのまま実装してください。特に`RunnablePathFrom`関数の中にある「`^[a-z]:\\`で始まるかどうか」というチェックは、**絶対に緩めたり削除したりしないでください**。これは以下のリスクを防ぐための重要な安全策です。

- ネットワーク上の共有フォルダ（`\\server\share\...`のような形式）経由での実行を防ぐ
- 相対パスによる意図しないファイルの実行を防ぐ

この判定に通った場合のみ、履歴項目を右クリックしたときのメニューに「このファイルを開く」または「このフォルダを開く」という項目が追加されます。実行可能なファイル（.exe、.bat等）を開こうとした場合は、必ず確認ダイアログを表示してから実行してください（設計書のコードに既に含まれています）。

## 厳守事項（地雷）

- **履歴（ClipHistory）に関するファイル書き込み処理は一切追加しないこと。** 今回書き込んで良いのは、ユーザーが明示的に「インポート」を選んだときの定型文ファイル（snippets.ini）だけです
- **`RunnablePathFrom`関数のパス判定を緩めないこと**。特に`^[a-z]:\\`の正規表現チェックは変更しないでください
- **`IniWrite`や`IniRead`関数を使わないこと**。文字コードの問題があるため、既存のコードと同じ流儀（`FileRead`/`FileAppend`をUTF-8明示で使う）を踏襲してください
- **既存の`LoadSnippets()`関数の基本的な動作（引数なしで呼んだ場合は従来通りsnippets.iniを読む）を壊さないこと**。パス引数を追加する際は、省略可能なオプション引数にしてください
- **ファイルを開く前の確認ダイアログ（実行可能ファイルの場合）を省略しないこと**
- **ビルドは`scripts/build.ps1`を使うこと**。Git BashからAhk2Exeを直接呼ぶと引数が壊れる既知の問題があります

## 完了したら確認すること

1. トレイメニューから「定型文ファイルを編集」を選ぶと、メモ帳でsnippets.iniが開くこと
2. 「設定フォルダを開く」でエクスプローラーが開くこと
3. 別のテキストファイルに「テスト=これはテストです」のような1行を書いて保存し、「定型文をインポート」でそのファイルを選択すると、送信サジェストの定型文リストに追加されること。既存の定型文と同じラベル名のものはインポートされない（重複しない）こと
4. 何かファイル（例えば`C:\Windows\System32\notepad.exe`のパス）をコピーしてクリップボード履歴に入れ、その履歴項目を右クリックすると「このファイルを開く」という選択肢が出て、選ぶと確認ダイアログの後にメモ帳が起動すること
5. 何かフォルダのパス（例えば`C:\Windows`）をコピーして同様に試すと「このフォルダを開く」でエクスプローラーが開くこと
6. 普通の文章（実在しないパス）をコピーした場合、右クリックしても「このファイルを開く」のような項目は出ないこと（誤動作の確認）
7. 既存の全機能（グローバル監視、削除、数字キー選択、ドラッグ移動等）が引き続き正常に動くこと（回帰確認）

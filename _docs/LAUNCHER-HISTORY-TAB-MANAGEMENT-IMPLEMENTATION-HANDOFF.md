# 実装ハンドオフ: ランチャー「クリップボード」タブ管理機能

設計は完了済み。このファイル1枚で着手できる。設計の全文は [`LAUNCHER-HISTORY-TAB-MANAGEMENT-DESIGN.md`](LAUNCHER-HISTORY-TAB-MANAGEMENT-DESIGN.md) 参照（読む場合はA節・C節・G節だけで足りる）。

## 発端

ユーザーから「定型文タブの右クリックメニュー（編集・削除・↑↓移動・新規登録・検索・ページ切り替え）が、クリップボード履歴タブには無い。実装してほしい」という要望。調査の結果、**そのメニューは実は別ウィンドウ「定型文の管理」のものでランチャー本体には無かった**という誤認が発端。ただし調査の過程で**実際のバグ**（後述）を発見した。

## 最重要: 実装前に必ず読むこと

**「この履歴を削除」は永続ストアに反映されず、アプリ再起動で削除項目が復活するバグがある**（`dist/soushin-suggest.ahk:3004` `DeleteHistoryAt`）。メモリの配列からは消えるが、`history-store.csv`は追記式（`AppendHistoryStore` 2558行）で削除に対応した書き直し処理が無い（71行の`HistStoreRewritePending`は宣言のみで未使用）。永続化は既定ONなので、ユーザーが「消したはず」の履歴（機密情報である可能性）が再起動後に蘇る。**これは新機能追加より先に直す不具合であり、他のどの管理機能追加よりも優先すること。**

## 検証（機械的な完了判定）

1. **削除の永続反映**: ランチャーで履歴を1件右クリック→削除→アプリを再起動→その項目が復活しないことを確認。`history-store.csv`を開き、削除した項目の行が無く、他の行は残っていることを確認。
2. **idxずれ対策**: メニュー表示中に別アプリで新規コピーを行い、その状態で「この履歴を削除」を押しても、メニューを開いたときに右クリックしていた項目が削除されること（新規コピーで先頭に積まれた別項目が誤って消えないこと）。
3. **コピーのみ**: 履歴（テキスト・画像それぞれ）を右クリック→「コピーのみ」→クリップボードに入るが貼り付け先には何も起きないこと。
4. **全削除確認**: トレイメニュー・ランチャー右クリックどちらから実行しても確認ダイアログが出て、「いいえ」でキャンセルできること。
5. **日時検索**: 検索欄に日付の一部（例: 過去に履歴を作った日の月日）を入力し、その日時の履歴がヒットすること。
6. **管理画面への導線**: ランチャーで検索語を入れた状態から「管理画面で履歴を見る」を押すと、定型文の管理ウィンドウが開き履歴タブが選択され、検索語が引き継がれていること。
7. **定型文タブの新メニュー**: 定型文タブで右クリック→「編集・削除（定型文の管理）」で管理ウィンドウが開くこと。

## 地雷（設計書G節から転記、特に重要なもの）

- **WS_EX_COMPOSITEDの再提案は絶対禁止**（過去に白化バグの原因として撤回済み）。本実装でGUIスタイルは一切触らない。
- **フッターロゴのアンカー計算（1631-1644行）に触れない**。ボタン行やフィルタバーを追加しない（設計書で明示的に却下済み）。
- **`LauncherLVCustomDraw`（372-374行）の不変条件を破らない**。描画状態を書き換えるAPIを呼ばない。
- **メニューのクロージャは行インデックス`idx`でなく要素参照`v`を束縛する**。メニュー表示中も監視は生きており、新規コピーで配列がずれるため。
- **MsgBox（確認ダイアログ）を出す前に必ず`CloseLauncher()`する**（`PromoteHistoryAt`と同じ既存の流儀）。
- **ストアの書き直しは全体書き直しでなく差分削除方式**。全体書き直しは`ClipHistLoadMax`（既定10000件）打ち切り分の古い行を巻き込んでデータロスする。
- **`SnipMgrTab.Value`への代入はChangeイベントを発火しない**（AHK v2仕様）。`SnipMgrHistRefresh()`を明示的に呼ぶこと。
- Windows注意: 検証用スクリプトに日本語コメントを書かない（Shift-JIS誤読地雷）。

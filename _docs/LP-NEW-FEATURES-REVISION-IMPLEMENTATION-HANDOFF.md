## 読む順
1. 上記設計書全体(特にB節の書き換え文言、C節の新セクションコピー、G節の地雷)
2. `index.html`のL2475(#mouse-flow終端)〜L2492(safety終端)付近を実地確認してから着手

## スコープ
- 新規セクション`#clipboard`を#mouse-flow直後に追加(設計書A・C節)
- safetyセクションを2カード構成に書き換え(設計書B節)

- UI・機能側のコード変更は一切なし(LPのコピー・マークアップのみ)

## 着手手順
1. 現在のブランチ`feature/clipboard-history-archive`で継続、またはLP専用の新規ブランチを切る
2. 設計書G-1〜G-2に従い`.safety`のDOM構造(icon+div)を確認してから2カード化
3. 設計書C節のコピーをそのまま使い、既存の`.flow-card`/`.step-card`相当のクラスを流用してカード4枚を作成
4. safetyカード1の主語限定とカード2の追加は**同一コミットで行う**(G-6、片方だけだと文言が矛盾する)
5. #faq追記3問(G-5)

## 機械的な完了判定
- [ ] `#clipboard`セクションが#mouse-flow直後・safety直前に存在する
- [ ] safetyカード1に「送信や自動操作は」という主語限定が入っている
- [ ] safetyカード2(グローバル監視・永続化の開示)が追加されている
- [ ] 「非永続」という語がLP本文に登場していない
- [ ] 「検疫」という語が見出しに使われていない(本文中の説明語としてのみ可)
- [ ] Clibor取込の説明が「CSVを選ぶだけ」であり「ドロップ」という語を使っていない
- [ ] 「設定ファイルを開く必要はありません」等の文が保存設定の文脈に限定されており、送信対応サイト追加(既存の設定ファイル方式)の説明と矛盾していない
- [ ] #faqに3問追記されている(Clibor移行/保存されるか/どのアプリのコピーも記録するか)
- [ ] ページ内リンク・既存id(`features`/`mouse-flow`/`price`/`faq`/`side-button`)に衝突や破損がない
- [ ] ブラウザでLPを開き、新セクションが崩れなく表示される(レスポンシブ含む)

## 地雷(再掲・最重要のみ)
- カード1の主語限定とカード2追加は同一コミットで行うこと(設計書G-6、最重要)
- 「設定ファイルの編集も不要です」(L2553)等の既存文言は触らない。今回の設定ウィンドウは保存設定限定の話で、送信対応サイト追加は今も設定ファイル方式(設計書G-4)
- サイドボタンの説明(L2410-2412, L2756)はクイックペーストに未言及という既知の差分がある。新セクションで起動方法に言及する場合は実装と照合すること(設計書G-8)

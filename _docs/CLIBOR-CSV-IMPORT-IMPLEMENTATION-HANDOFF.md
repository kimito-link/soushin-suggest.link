# 実装ハンドオフ: CliborエクスポートCSV取り込み機能

設計本体は [`CLIBOR-CSV-IMPORT-DESIGN.md`](CLIBOR-CSV-IMPORT-DESIGN.md)。

## 読む順
1. 上記設計書全体（特にC節のコード断片、G節の地雷）
2. `dist/soushin-suggest.ahk`の`ParseCsv`、`LoadSnippetsCsv`、`ImportSnippets`、`LoadSnippets`（既存のCSV/ini処理パターン）

## スコープ（MVP、設計書E節）
- ヘッダ行自動判別によるClibor形式CSVの取り込み
- ラベル自動生成＋重複時の一意化サフィックス
- グループ名プレフィックス（2種類以上ある場合のみ）
- UI追加なし（既存のインポート導線をそのまま使う）

## 着手手順
1. 新規ブランチ（例: `feature/clibor-csv-import`）
2. 設計書C-1〜C-3のコード断片を追加（`TryLoadCliborCsv`/`CliborHeaderOk`/`CliborLabelBase`）
3. `ImportSnippets`にC-4の分岐を追加
4. バージョン文字列更新
5. ビルド: `scripts/build.ps1`経由、ビルド前にプロセス終了

## 機械的な完了判定

**重要**: テストには実際のCliborエクスポートCSVが必要。テスト用のダミーCSVを作る場合も、実在するパスワード等の機密情報は絶対に含めないこと。

- [ ] ビルドがエラーなく完了
- [ ] Cliborエクスポート形式のCSV（`定型文グループ,定型文,メモ,ホットキー`ヘッダ）を「定型文の管理」の取込機能で選ぶと、文字化けせずに正しく取り込まれる
- [ ] メモ列が空の項目は、本文の先頭20文字がラベルになる
- [ ] メモ列がある項目は、メモがラベルになる
- [ ] 同一本文が複数行ある場合、ラベルが`(2)``(3)`...のように一意化される（データが消えずに全件取り込まれることを確認）
- [ ] 同じCSVファイルをもう一度取り込んでも、定型文が倍増しない（冪等性の確認）
- [ ] グループが1種類しかない場合、ラベルにグループ名プレフィックスが付かない
- [ ] グループが2種類以上ある場合、ラベルに`グループ名/`プレフィックスが付く
- [ ] 既存の自社CSV形式（`label,body`）の取込が壊れていないこと（回帰確認）
- [ ] 取込後のラベルに`=`や先頭`;`/`[`が含まれず、アプリ再起動後も定型文が正しく読み込まれること

## 地雷（再掲・最重要のみ）
- 既存の`ImportSnippets`は重複ラベルを無言スキップする。ローダー内で一意化を完結させないとデータが消える（G-1、最重要）
- `FileRead`のエンコーディング指定（`"CP932"`）を明示し忘れると文字化けする。ただしヘッダ不一致で従来パスに落ちるため「化けたまま取り込む」事故は起きない設計（G-5）
- ラベル生成時に`=`や行頭`;`/`[`をサニタイズしないと、次回起動時にsnippets.iniが正しく読めなくなる（G-3）

## 転記元（設計書からコピーする実在パス）
- `dist/soushin-suggest.ahk` — 唯一の実装対象ファイル
- `scripts/build.ps1` — ビルドスクリプト（変更不要）

## 完了後
- コミットメッセージは設計書へのリンクと変更概要を含める
- **コミット前に、テストで使った実データ由来のCSVファイルやその複製がリポジトリに含まれていないことを必ず確認する**（機密情報混入防止）
- 非自明だった実装判断（設計書との差分が生じた場合）があればメモリに記録

# 実装ハンドオフ: 定型文管理ウィンドウへの履歴タブ追加

設計本体は [`SNIPPET-MANAGER-HISTORY-TAB-DESIGN.md`](SNIPPET-MANAGER-HISTORY-TAB-DESIGN.md)。

## 読む順
1. 上記設計書全体（特にC節のコード断片とG節の地雷、特にG-1の座標シフト表）
2. `dist/soushin-suggest.ahk`の`ShowSnippetManager()`（現在の実装、Tab3挿入前の座標を確認）

## スコープ（MVP、設計書E節）
- Tab3化（定型文/履歴）、履歴タブに検索・一覧・全文プレビュー・コピー
- 入れないもの: ソート切替、履歴の個別削除、ペースト機能

## 着手手順
1. 現在のブランチ`feature/snippet-manager-window`で継続、またはコミット後に新ブランチ
2. 設計書C節のコード断片を`ShowSnippetManager()`に統合。既存12コントロールのy座標を+26シフト（G-1表）
3. `SnipMgrHistRefresh`/`SnipMgrHistOnSelect`/`SnipMgrHistCopy`/`SnipMgrTabChanged`を追加
4. グローバル宣言に新設6変数を追加し忘れない（G-9）
5. バージョン文字列更新（2箇所）
6. ビルド: `scripts/build.ps1`経由、ビルド前にプロセス終了

## 機械的な完了判定
- [ ] ビルドがエラーなく完了
- [ ] 「定型文の管理」を開くと[定型文][履歴]タブが見える、既定は定型文タブ
- [ ] 履歴タブで実際のコピー履歴が新しい順に表示される
- [ ] 検索ボックスに文字を打つと絞り込まれる
- [ ] 行選択で全文プレビューに表示される
- [ ] 「クリップボードへコピー」またはダブルクリックでコピーされる（別アプリにCtrl+Vして確認）
- [ ] 履歴タブ表示中、CSV出力/取込/追加/保存/削除ボタンが一切見えない・操作できない
- [ ] 定型文タブに切り替えると元通り操作できる
- [ ] 既存のクイックペーストランチャー（サイドボタン長押し）の履歴タブ・ワンキーペーストが壊れていない
- [ ] 履歴はディスクに一切書き込まれない（アプリ終了で消えることを確認）

## 地雷（再掲・最重要のみ）
- 座標シフトの取りこぼし（G-1）。12箇所全て機械的に+26する
- `UseTab()`の閉じ忘れ（G-2）
- コピー操作が履歴自身の並び順を変える可能性への対処として、`SnipMgrHistRows`は要素参照で持つ（G-3）
- `NoSort NoSortHdr`を履歴LVにも付ける（G-6）

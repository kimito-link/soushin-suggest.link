# 実装ハンドオフ: 「操作の流れは、ひと筆書き。」セクション追加

このファイル1枚を読めば実装に着手できる。設計の背景・会議の議論・捨てた案は
[`MOUSE-FLOW-CLARITY-DESIGN.md`](MOUSE-FLOW-CLARITY-DESIGN.md) 参照（着手には必須ではない）。

## スコープ（MVPのみ）

- `index.html` に新規セクション `<section id="mouse-flow">` を1つ追加する
- HTML本体は設計書 C-1、CSSは設計書 C-2 をそのまま使う（コピペでよい、文言も確定済み）
- Phase 2（なぞりアニメーション演出 C-3）・Phase 3（キャラ画像）はやらない。今回はテキストのみ

## 着手手順

1. ブランチを切る: `git checkout -b feature/mouse-flow-section`
2. 挿入位置を特定:
   ```
   grep -n '効かせる場所は、選んでいます' index.html
   ```
   ヒットした行の**直前**にある `<section` 開始タグを探し、その直前に設計書C-1のHTMLを挿入する
   （`</section>`〜次の`<section id="safety">`の間、既存のsectionを壊さないよう注意）。
3. CSS挿入位置を特定:
   ```
   grep -n 'steps — 導入の流れ' index.html
   ```
   ヒットした行の直前に設計書C-2のCSSを追記する。
4. 設計書C-1・C-2の内容をそのままEdit toolでindex.htmlに書き込む（改変不要、確定文言）。

- [ ] ブラウザでLPを開き、`#features`セクションの直後・`#safety`セクションの直前に新セクションが表示される
- [ ] モバイル幅（375px）で縦積み1カラム、デスクトップ幅（1280px）で2カラムになる（DevToolsでリサイズ確認）
- [ ] `#side-button`へのアンカージャンプ（ヘッダーメニューか既存リンクから）が、固定ヘッダーに隠れず正しい位置に着地する（過去に壊れた実績があるための回帰確認）
- [ ] コンソールエラーが出ていない

## 地雷（実装時に必ず守ること）

- index.htmlは655KB超の単一ファイル。**Readツールで全文読みしない**。`grep -n`で対象行を特定してからEdit tool使用
- HTML/CSSは設計書の確定文言をそのまま使う。改変する場合も「なぞる＝コピー完了」「長押し＝貼り付けて送信」という**操作＝機能の等式見出し**の形は崩さない（位置ベースの説明への逆戻り厳禁）
- キャラ画像は使わない（テキストラベルのみ）。画像を追加したくなっても、既存の`<img>`行をsedでコピーしない（ラッパー巻き込み事故の前科あり）
- SVGを新規に描かない。矢印はCSS疑似要素の文字のみ

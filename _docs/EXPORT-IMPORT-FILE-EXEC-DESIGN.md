## B. 要望J（インポート・エクスポート）の判定と設計

### B-1. 履歴: インポート却下・エクスポート却下

会議の警告にはこう応える — **警告は正しく、しかも既存コードがすでに答えを持っている。**

このアプリの安全設計は「履歴はメモリのみ」という一点に多くを預けている。それはコードの随所に現れている: グローバル変数のコメント（`メモリのみ・非永続`）、`MaybeDropAutoCleared()`（パスワードマネージャの自動クリアに追従して履歴からも消す）、`ClipHasIgnoreFormat()`、`ClipExcludeExes`。これらは全部「拾ってしまった秘密は、消えるときに一緒に消える」ことを前提にした防御だ。履歴がディスクに残った瞬間、この防御網は全部**過去に向かって無効化**される — 45秒で自動クリアしても、エクスポートファイルには残っているのだから。

- **インポート却下の理由**: インポートとはすなわち「起動時に外部ファイルから履歴を復元する」経路であり、非永続の定義そのものと矛盾する。暗号化しても「ディスク上に履歴が存在する」事実は変わらず、鍵管理という新しい負債（603行アプリには過大）だけが増える。
- **エクスポート却下の理由**: ファイル書き出しは上記の通り。「クリップボード経由の一括コピー」案（ディスク非経由）も検討したが却下する。最大30件×各10万字=数MBの塊に、ユーザーが忘れている秘密が混ざったまま1回の誤ペーストで全部漏れる。既存設計の美点は「1件ずつ、意識的に選んで取り出す」ことを強制している点（数字キーで1件ペースト、右クリックで1件昇格）で、一括コピーはそれを壊す。
- **「残したい」への正規の答え**: `PromoteHistoryAt()`がすでにある。履歴の中で本当に価値のある1件は、名前を付けて定型文に昇格させる — これが**唯一の、意識的な永続化ゲート**であり、この設計書はそれを今後も唯一であり続けさせる。

ユーザーに聞かれたときの説明文もここで確定しておく:
> 「履歴はアプリを閉じると消える設計です（パスワード等を拾ってしまっても残らないための安全機能）。残したいものは右クリック→『定型文に登録』してください。定型文はファイルなので持ち出せます。」

### B-2. 定型文: エクスポート＝ファイルコピー、インポート＝追記マージ

`snippets.ini`は既にディスク上のプレーンなUTF-8ファイルであり、`PromoteHistoryAt()`が`FileAppend`で追記している。**新たなリスクはゼロ**。よって:

**エクスポート**: 専用機能を書かない。snippets.iniをコピーすることがエクスポートである。問題は非エンジニアのユーザーが`A_ScriptDir`を見つけられないことだけなので、トレイメニューに2項目足す:

- 「定型文ファイルを編集 (snippets.ini)」— メモ帳で開く（無ければ空で作ってから開く）
- 「設定フォルダを開く」— explorerでA_ScriptDirを開く（sites.iniの編集導線にもなる副次効果あり）

**インポート**: FileSelectでiniを選ばせ、既存の`LoadSnippets()`パーサをそのまま流用して読み、**既存ラベルと重複しないものだけ追記**する。上書きマージはしない（自分の定型文を壊す事故の方が怖い）。`LoadSnippets()`にパス引数を足すだけでパーサの二重実装を避けられる。

## C. 要望K（フルスペックのファイル検索）の判定

**却下。** 会議の結論を全面的に支持し、これを最終確定とする。

1. **行数**: インデックス構築・インクリメンタル検索UI・PATH解決・ランキングで数千行。603行の全体が「1ファイル読めば全部わかる」規模であることが本製品の保守性の実体であり、それを壊す。
2. **ブランド**: 本製品は「マウスだけで完結」（なぞってコピー・長押し送信・サイドボタン）。文字入力を前提とする検索ボックスはブランド軸の反対側にある。
3. **競合**: PowerToys Run / Flow Launcherが無料で存在する。買い切り製品が勝てない土俵に登る理由がない。
4. **前例**: 過去に検索機能を却下して`run:`プレフィックスに落とした判断は正しく機能している。同じ判断を繰り返す。

## D. 発散案（クリップボード文脈限定の実行）の評価と採否

**採用する。ただし会議案からさらにスコープを削る。**

会議案の「exe名らしき文字列」検出は却下する。「らしき」の解決にはPATH探索・拡張子補完・曖昧マッチが要り、それは小さな検索エンジンであり、Kの却下理由がそのまま再適用される。さらに「chrome」という単語をコピーしただけの履歴に実行ボタンが生えるのは誤爆源になる。

採用するのは次の一点だけ:

> **履歴項目のテキストが、実在するローカルのドライブレター絶対パス（ファイルまたはフォルダ）であるときに限り、既存の履歴右クリックメニューに「開く」を追加する。**

- 検索しない。推測しない。`FileExist()`が真を返す文字列だけが対象。判定は右クリックされた瞬間の1回だけで、リスト表示には何の印も出さない（一覧に▶を出すと誤クリック面が増えるため。定型文タブの▶はユーザーが自分で書いたrun:なので事情が違う）。
- エクスプローラーでパスをコピー（Shift+右クリック「パスのコピー」や、アドレスバーコピー）→ サイドボタン長押し → 右クリック「開く」、が想定フロー。マウス中心のブランド軸とも整合する。

**`run:`定型文との住み分け**（明確に定義する）:

| | `run:`定型文 | 履歴の「開く」 |
|---|---|---|
| 性質 | 永続・ユーザーが自分で登録した厳選ランチャー | 一時的・さっきコピーしたパスのその場実行 |
| 起動導線 | 定型文タブ＋数字キー（ワンアクション） | 履歴右クリック→メニュー（ツーアクション） |
| 確認 | なし（本人が書いたもの） | 実行可能拡張子のみ確認ダイアログあり |
| 昇華経路 | — | 何度も使うパスは「定型文に登録」でrun:化する |

「単発は履歴から、常用は昇格してrun:へ」— 既存の昇格機能が2機能を接続するので、新しい概念をユーザーに教える必要がない。

### E-2. 定型文インポート（LoadSnippetsのパス引数化＋追記マージ）

`LoadSnippets()`の先頭2行だけ変更:

```autohotkey
LoadSnippets(path := "") {
    items := []
    if (path = "")
        path := A_ScriptDir . "\snippets.ini"
    ; 以下は現行のまま
```

```autohotkey
; 別ファイルから定型文を追記マージ。既存ラベルと重複するものは飛ばす。
; 書き込みはPromoteHistoryAtと同じ流儀（IniWrite不使用・UTF-8明示のFileAppend）
ImportSnippets(*) {
    f := FileSelect(1, , "取り込む定型文ファイルを選択", "定型文ファイル (*.ini; *.txt)")
    if (f = "")
        return
    try if (FileGetSize(f) > 1024 * 1024) {
        Flash("ファイルが大きすぎます（1MB超）", 1800)
        return
    }
    incoming := LoadSnippets(f)
    if (incoming.Length = 0) {
        Flash("取り込める定型文が見つかりませんでした（ラベル=本文 の形式）", 2000)
        return
    }
    have := Map()
    for s in LoadSnippets()
        have[s.label] := 1
    path := A_ScriptDir . "\snippets.ini"
    added := 0
    try {
        nl := (FileExist(path) && !RegExMatch(FileRead(path, "UTF-8"), "\R$")) ? "`n" : ""
        for s in incoming {
            if have.Has(s.label)
                continue
            FileAppend(nl . s.label . "=" . StrReplace(s.value, "`n", "\n") . "`n", path, "UTF-8")
            nl := "", added++
        }
        Flash(added . " 件を取り込みました（重複 " . (incoming.Length - added) . " 件はスキップ）", 2000)
    } catch as e {
        Flash("インポートに失敗しました: " . e.Message, 2000)
    }
}
```

（次回`ShowLauncher()`が`LoadSnippets()`を毎回読み直す設計のおかげで、リフレッシュ処理は不要。）

### E-3. 履歴の「開く」（判定関数＋実行関数＋メニュー配線）

```autohotkey
; 履歴テキストが「実在するローカルのドライブレター絶対パス」のときだけ正規化して返す。
; UNC(\\server)・相対パス・複数行・存在しないパスはすべて空を返す(fail-closed)。
RunnablePathFrom(text) {
    p := Trim(text, " `t`r`n`"'")
    if InStr(p, "`n") || StrLen(p) > 500
        return ""
    if !RegExMatch(p, "i)^[a-z]:\\")
        return ""
    return FileExist(p) ? p : ""
}

OpenHistoryPath(p) {
    CloseLauncher()
    if InStr(FileExist(p), "D") {
        try Run('explorer.exe "' . p . '"')
        return
    }
    if RegExMatch(p, "i)\.(exe|bat|cmd|com|ps1|vbs|js|wsf|msi|scr|lnk)$") {
        if (MsgBox("これはプログラムです。実行しますか？`n`n" . p, "履歴から実行", "YesNo Icon! Default2") != "Yes")
            return
    }
    try Run('"' . p . '"')
    catch
        Flash("開けませんでした: " . p, 1800)
}
```

`LauncherContextMenu()`の履歴メニュー構築部に2行追加:

```autohotkey
        if (p := RunnablePathFrom(ClipHistory[idx].text))

        m.Add("定型文に登録", (*) => PromoteHistoryAt(idx))
```

## G. 捨てた案と理由

1. **履歴のファイルエクスポート**: 非永続性の崩壊。B-1の通り。
2. **履歴の暗号化永続化**: 鍵管理という新負債＞得られる価値。603行アプリのスコープ外。
3. **クリップボード経由の履歴一括コピー**: ディスク非経由という工夫は評価するが、「1件ずつ意識的に取り出す」既存設計を壊し、誤ペースト1回で全件漏洩する塊を作る。昇格機能で十分。
4. **フルスペックのファイル検索**（要望K）: C章の通り。数千行・ブランド軸衝突・PowerToys Runと競合。
5. **「exe名らしき文字列」の曖昧検出**（発散案の原形）: 「らしき」の解決はPATH探索＝小さな検索エンジンであり、Kの却下理由が再適用される。実在絶対パス限定に削った。
6. **履歴リストへの▶マーク表示**: 誤クリック面を増やす。右クリックメニュー限定に留めた。
7. **URL（http/https）を「ブラウザで開く」**: 実装は3行だが、URLはペーストすれば済む上、フィッシングURLをワンクリック実行可能にする面が増える。必要の声が出てから再検討。
8. **定型文の上書きマージ／エクスポートUI**: 上書きは自分の定型文を壊す事故源。エクスポートは「ファイルをコピーする」以上の何かを作る理由がない。

## H. 地雷と回避策

1. **UNCパス実行**（`\\server\share\evil.exe`）: リモートコード実行の入口になる。`RunnablePathFrom`が`^[a-z]:\\`を要求することで構造的に遮断済み。**この正規表現を緩めないこと。**
2. **相対パスの解決先**: `FileExist`は相対パスをA_WorkingDir基準で解決するため、「たまたま存在するファイル名」が実行可能になりうる。同じくドライブレター必須で遮断済み。
3. **引数付きコマンド文字列**（`foo.exe -x`）: `FileExist`が文字列全体で判定するため自然に偽になる＝引数付き実行は構造的に不可能。この性質を壊す「気の利いた」引数分離を後から足さないこと。
4. **メニューコールバックのクロージャ**: `idx`/`p`はメニュー構築時に確定させてからクロージャに渡す。メニュー表示中に履歴が変わっても古い値で動くのは既存挙動と同じ許容範囲。
5. **メニュー表示中の誤クローズ**: 既存の`SetTimer(CheckLauncherFocus, 0)`の内側にメニュー項目を足すだけなので新規対処は不要。ただし`OpenHistoryPath`のMsgBoxは必ず`CloseLauncher()`の後に出す（AlwaysOnTopのGUIと確認ダイアログの前後関係が壊れるため）。
6. **snippets.iniの文字コード**: 既存の掟どおりIniRead/IniWriteは使わない（非ASCIIキー誤読の既知の罠）。インポートも`FileRead(f, "UTF-8")`経由（`LoadSnippets`流用で自動的に守られる）。BOM付きUTF-8はAHKのFileReadが吸収する。
7. **インポートファイルが巨大・バイナリ**: 1MB上限チェックで遮断。バイナリはパーサが`=`行を見つけられず0件になるだけで無害。
8. **`PromoteHistoryAt`のラベル正規化との整合**: 昇格時はラベルから`=[];`を剥ぐ。インポートは既にファイル形式になっているものを読むためパーサ通過＝正規化済みとみなしてよい。二重に剥がない。
9. **将来の誘惑への防波堤**: 「履歴もiniに書けば再起動で消えないのに」という提案は今後も必ず出る。ClipHistoryのグローバル宣言コメントを次のように強化しておくこと: `; {text,time}の配列・メモリのみ・非永続 — 永続化禁止(唯一の安全特性。エクスポートも不可。経緯は_docs参照)`。

# 実装ハンドオフ: クリップボード履歴の画像対応

設計本体は [`CLIPBOARD-IMAGE-HISTORY-DESIGN.md`](CLIPBOARD-IMAGE-HISTORY-DESIGN.md)。

## 読む順
1. 上記設計書全体（特にC節のコード断片、G節の地雷、「設計の核となる判断」の節）
2. `dist/soushin-suggest.ahk`の`ClipChanged`/`CaptureClip`（L774-804付近）、`PushClipHistory`（L190付近）、`PasteHistoryAt`（L737付近）、`ClipHasIgnoreFormat`/`ClipSourceExcluded`（既存のクリップボード判定パターン）

## スコープ（MVP、設計書E節）
- 画像コピー→プレースホルダーで履歴に載る→選ぶと貼り付く、の一本道
- 安全装置: ユーザー操作フィルタ・除外元判定・件数/サイズ上限・fail-closedなクリップボード開閉
- 入れないもの: サムネイル表示、画像の重複排除、自動クリア検知の画像対応

## 着手手順
1. 新規ブランチ（例: `feature/clipboard-image-history`）
2. 設計書C-1〜C-5のコード断片を実装。既存関数への分岐追加（D節の表）を機械的に適用
3. `ClipHistory`要素に`type`フィールドを持たせる変更は`PushClipHistory`1箇所の追加で足りるが、**既存の重複判定ループに`v.type = "text"`ガードを追加し忘れない**（G節で言及される画像/テキスト誤爆の防止）
4. `sites.ini`の`[clipboard]`パーサに`imagemax=`/`imagemaxmb=`の2キーを追加
5. バージョン文字列更新（2箇所）
6. ビルド: `scripts/build.ps1`経由、ビルド前にプロセス終了

## 機械的な完了判定
- [ ] ビルドがエラーなく完了
- [ ] スクリーンショット（PrintScreen、Win+Shift+S等）をコピーすると、ランチャーの履歴タブに「📷 画像 WxH (サイズMB)」のような行が現れる
- [ ] その行を選択（クリックまたは数字キー）すると、別アプリ（ペイント等）に実際に画像が貼り付けられる
- [ ] 「定型文の管理」ウィンドウの履歴タブでも同様に画像行が表示・検索・コピーできる
- [ ] 画像を「定型文に登録」しようとしても登録できない（メニューに出ない、またはガードで弾かれる）ことを確認
- [ ] 画像を6枚以上連続コピーすると、画像専用上限（既定5件）で古いものから消えることを確認
- [ ] 36MBを超える巨大画像をコピーしても履歴に載らない（サイズ上限で弾かれる）ことを確認できれば理想（大きな画像の用意が難しければスキップ可）
- [ ] クリップボード監視を一時停止した状態で画像をコピーしても履歴に載らない
- [ ] アプリを終了すると画像履歴も消える（非永続の確認）。ディスクに一時ファイルが残っていないことを確認（`%TEMP%`を目視）
- [ ] 既存のテキスト履歴・定型文機能に回帰がないこと

## 地雷（再掲・最重要のみ）
- `OpenClipboard`したら必ず`try { … } finally DllCall("CloseClipboard")`で閉じる。閉じ忘れはOS全体のクリップボードを壊す最悪級の事故（G-2）
- `OnClipboardChange`コールバック内で直接`OpenClipboard`しない。既存の`SetTimer(…, -120)`デバウンスパターンを踏襲する（G-3）
- `SetClipboardData`成功後は`GlobalFree`しない（二重解放でクラッシュ、G-5）
- `SelfClipTick`は`EmptyClipboard`より**前**に立てる（自己書き込みループ防止、G-6）
- HBITMAP/HGLOBALハンドルを履歴に持ち越さない。取得したその場で`Buffer`にコピーする（設計の核）
